🎯 **What:** The testing gap addressed
Tested the `get_thread_connection` and `close_thread_connection` functions from `src/extracao/extrair_dados_cnpj.py` which had no tests previously.

📊 **Coverage:** What scenarios are now tested
1. **Creation:** Evaluates if a new thread connection is instantiated and appropriately assigned when it doesn't already exist.
2. **Reuse:** Ensures that sequential calls on the same thread reuse the cached connection, averting redundant connection constructions.
3. **Thread Isolation:** Confirms that individual threads manage their distinct connection instances, effectively avoiding race conditions and connection crosstalk.
4. **Closing (Happy path & Exception handling):** Verifies that `close_thread_connection` efficiently terminates active connections and handles scenarios where connection close raises exceptions gracefully or no connection was established to begin with.

✨ **Result:** The improvement in test coverage
`src/extracao/extrair_dados_cnpj.py` now has robust, deterministic unit tests for its crucial multi-threading database connection management aspect, shielding it from potential future threading regressions.
